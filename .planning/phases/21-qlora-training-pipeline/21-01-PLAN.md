**trainer.py** (NEW file) — Create the `train_qlora()` orchestrator function. ALL GPU imports MUST be deferred inside the function body per INFRA-05. This function orchestrates the entire training pipeline.

```python
def train_qlora(
    session_id: str,
    adapter_id: str,
    output_dir: str,
    *,
    base_model_id: str | None = None,
    task_type: str = "code-gen",
    rank: int = 64,
    alpha: int = 128,
    epochs: int = 3,
    learning_rate: float = 2e-4,
) -> str:
```

Implementation steps inside the function body:
1. Deferred imports: `from datasets import Dataset`, `from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig`, `from trl import SFTConfig, SFTTrainer`, `import torch`
2. Import from model_training: `from model_training.peft_utils import build_qlora_config`, `from model_training.trajectory import load_trajectory, format_for_sft`
3. Read RUNE_BASE_MODEL env var inside function body (default "Qwen/Qwen2.5-Coder-7B-Instruct") if base_model_id is None. Use `os.environ.get("RUNE_BASE_MODEL", "Qwen/Qwen2.5-Coder-7B-Instruct")`.
4. Load trajectory via `load_trajectory(session_id)`, format via `format_for_sft(trajectory)`. If messages is empty, raise `ValueError(f"Trajectory {session_id} is not successful or has no SFT messages")`.
5. Create dataset: `Dataset.from_list([{"messages": messages}])`
6. Build BitsAndBytesConfig: load_in_4bit=True, bnb_4bit_quant_type="nf4", bnb_4bit_compute_dtype=torch.bfloat16, bnb_4bit_use_double_quant=True
7. Load model: `AutoModelForCausalLM.from_pretrained(model_id, quantization_config=bnb_config, device_map="auto")`
8. Load tokenizer: `AutoTokenizer.from_pretrained(model_id)`
9. Build LoRA config: `build_qlora_config(rank=rank, alpha=alpha, target_modules=["q_proj", "v_proj"], dropout=0.1)`
10. Create SFTConfig with: output_dir, num_train_epochs=epochs, learning_rate=learning_rate, warmup_ratio=0.03, lr_scheduler_type="cosine", bf16=True, per_device_train_batch_size=1, gradient_accumulation_steps=4, save_strategy="no", logging_steps=1, report_to="none", eval_strategy="no"
11. Create SFTTrainer: model=model, args=training_args, train_dataset=dataset, peft_config=lora_config, processing_class=tokenizer
12. Call trainer.train()
13. Call trainer.save_model(output_dir) — saves adapter_model.safetensors + adapter_config.json only (PEFT-aware save)
14. Return output_dir

Add a second helper function for the full pipeline including registry storage:

```python
def train_and_register(
    session_id: str,
    adapter_id: str,
    *,
    base_model_id: str | None = None,
    task_type: str = "code-gen",
    rank: int = 64,
    alpha: int = 128,
    epochs: int = 3,
    learning_rate: float = 2e-4,
    database_url: str | None = None,
) -> str:
```

This function:
1. Creates adapter output dir: `~/.rune/adapters/{adapter_id}/` (create parents, exist_ok=True). Read RUNE_ADAPTER_DIR env var inside function body, default to `Path.home() / ".rune" / "adapters"`.
2. Calls train_qlora() with the adapter dir as output_dir
3. Builds an AdapterRecord:
   - id=adapter_id, version=1, task_type=task_type, base_model_id (from env or param), rank=rank
   - created_at=datetime.now(tz=timezone.utc).isoformat()
   - file_path=str(adapter_dir), file_hash=sha256 of adapter_model.safetensors, file_size_bytes from stat
   - source="qlora", session_id=session_id
4. Creates AdapterRegistry with engine from database_url (or RUNE_DATABASE_URL env var, default "sqlite:///~/.rune/rune.db")
5. Calls registry.store(record)
6. Returns adapter_id

Use `from __future__ import annotations` at top. Module-level imports: only stdlib (os, hashlib, pathlib, datetime). Everything else deferred inside function bodies.
