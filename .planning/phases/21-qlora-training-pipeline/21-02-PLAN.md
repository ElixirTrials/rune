```python
"""In-memory training job status tracking.

Module-level JOB_STORE dict shared across all request handlers.
State is lost on service restart — acceptable for single-user local MVP.
"""
from __future__ import annotations

from dataclasses import dataclass
from typing import Optional

JOB_STORE: dict[str, JobStatus] = {}


@dataclass
class JobStatus:
    """Training job status tracker."""
    job_id: str
    status: str  # "queued" | "running" | "completed" | "failed"
    adapter_id: Optional[str] = None
    error: Optional[str] = None
```

**routers/training.py** — Replace 501 stubs with real implementations for POST /train/lora and GET /jobs/{job_id}. Keep POST /train/hypernetwork as 501 stub (Phase 22).

POST /train/lora implementation:
1. Import `uuid` at top (stdlib). Import `BackgroundTasks` from fastapi.
2. Import `from training_svc.jobs import JOB_STORE, JobStatus` at top (no GPU deps).
3. Generate job_id and adapter_id (or use request.adapter_id if provided) via `str(uuid.uuid4())`.
4. Create JOB_STORE entry BEFORE adding background task: `JOB_STORE[job_id] = JobStatus(job_id=job_id, status="queued", adapter_id=adapter_id)`.
5. Add background task `_run_training_job` with params: job_id, session_id, adapter_id, task_type, rank, epochs, learning_rate.
6. Return `JSONResponse(content={"job_id": job_id, "status": "queued"}, status_code=200)`.
7. Function signature: `async def train_lora(request: LoraTrainingRequest, background_tasks: BackgroundTasks) -> JSONResponse:`

`_run_training_job` helper (module-level function, not async):
1. Update JOB_STORE[job_id].status = "running"
2. Try: import `from model_training.trainer import train_and_register` INSIDE the function body (deferred GPU import per INFRA-05)
3. Call train_and_register(session_id=session_id, adapter_id=adapter_id, task_type=task_type, rank=rank, epochs=epochs, learning_rate=learning_rate)
4. On success: JOB_STORE[job_id].status = "completed"
5. On exception: JOB_STORE[job_id].status = "failed", JOB_STORE[job_id].error = str(e)

GET /jobs/{job_id} implementation:
1. Look up job in JOB_STORE. If not found, raise HTTPException(status_code=404, detail=f"Job {job_id} not found").
2. Return JSONResponse with job.job_id, job.status, job.adapter_id, job.error.

Import HTTPException from fastapi at top.
