<objective>
Implement `apply_functional_lora`, a class-based context manager that patches transformer attention projection modules with F.linear forward functions carrying live hypernetwork tensor graph nodes, preserving autograd continuity through A and B matrices back to the hypernetwork head.

Purpose: Phase 29's training loop calls `with apply_functional_lora(base_model, lora_dict, hc):` each step. The LoRA A/B tensors produced by `HyperLoRA.generate_weights()` must carry gradients back through the hypernetwork so the head parameters update. PEFT cannot be used (it severs the autograd graph). This context manager is the entire gradient path from loss to hypernetwork.

Output:
- `libs/model-training/src/model_training/d2l_lora.py` — production implementation
- `libs/model-training/tests/test_d2l_lora.py` — tests verifying LORA-01 / LORA-02 / LORA-03
- Updated `__init__.py` comment block documenting `apply_functional_lora` import path
</objective>

    Helper `_make_lora_forward(W, bias, A, B, scale)`:
    - Defers `import torch.nn.functional as F  # noqa: PLC0415` inside the function body
    - `W_detached = W.detach()`, `bias_detached = bias.detach() if bias is not None else None`
    - Returns closure `patched_forward(x)`:
      ```
      base_out = F.linear(x, W_detached, bias_detached)
      lora_Ax = F.linear(x, A)           # (..., r)
      lora_out = F.linear(lora_Ax, B.t()) * scale   # (..., d_out)
      return base_out + lora_out
      ```

    Class `_FunctionalLoRAContext`:
    - `__init__(self, model, lora_dict, hc)` — stores all three, initializes `_saved_forwards: dict[str, Callable[..., Any]] = {}`

Iterates `model.named_modules()`, finds matching short names, extracts layer_pos, validates shapes, saves and replaces `.forward`. Returns self.
    - `__exit__` — `try/finally` loop restores every entry in `_saved_forwards`. Returns `None` (does NOT suppress exceptions).
