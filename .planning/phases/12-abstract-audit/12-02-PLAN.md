<task type="auto">
  <name>Task 3: Final build gate — uv run mkdocs build --strict exits 0</name>
  <files></files>
  <action>
Run the final build gate:

```bash
uv run mkdocs build --strict 2>&1
echo "Exit code: $?"
```

The command must print no warnings and exit 0. Warnings come in these forms; each has a specific fix:

**`WARNING - Doc file 'article/abstract.md' is not found in the docs_dir`**
→ `docs/article/abstract.md` was not created. This is a Plan 12-01 failure. Re-read Plan 12-01 Task 1 output and create the file.

**`WARNING - [...] docs/article/abstract.md is not included in the 'nav' configuration`**
→ The nav entry was not added to mkdocs.yml. Add `- Abstract: article/abstract.md` between Overview and Background in the Scientific Article nav section.

**`WARNING - [...] anchor 'X' not found`**
→ A link in index.md or abstract.md uses a heading anchor that does not exist. Check the link target heading — MkDocs lowercases and hyphenates heading text. Fix the link to match the generated anchor.

**`WARNING - [...] absolute_links`**
→ A link in index.md or abstract.md uses an absolute path. Convert to relative path.

Do NOT run `mkdocs build` without `--strict`. Only `--strict` surfaces the warnings that matter. If the build exits non-zero, read the full error output before making changes.

If fixes are required, commit each changed file:
```bash
git add <file>
git commit -m "fix(12-02): resolve mkdocs build warning in <file>"
```

After all fixes, re-run `uv run mkdocs build --strict` and confirm exit 0.
