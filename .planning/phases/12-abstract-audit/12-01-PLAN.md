```markdown
## Abstract

```

The abstract text must cover all four required elements in order, targeting 240–260 words total inside the div:

**Problem (~45 words):** Local coding agents (Aider, Cursor, Claude Code) operate within fixed context windows and lose procedural knowledge between sessions — architectural decisions, recurring bug patterns, project-specific idioms. Each session begins from zero. No mechanism exists to accumulate and retrieve session-derived knowledge as persistent, composable model parameters.

**Approach (~55 words):** Rune proposes parametric episodic memory for local coding agents: each coding session produces a write-once LoRA adapter encoding procedural knowledge as parameter deltas in the base model's weight space. Adapters are composable, non-destructive, and reversible — enabling selective retrieval without overwriting previously acquired knowledge. This places Rune in the composable weight-space category of memory strategies, distinct from RAG (token-space) and full fine-tuning (destructive weight-space).

**Mechanism (~75 words):** The proposed mechanism extends Doc-to-LoRA — a hypernetwork architecture validated on document question-answering — to code execution trajectories: sequences of generation attempts, execution results, and reflection steps. The hypernetwork maps trajectory inputs to LoRA adapter weights in a single forward pass. This trajectory modality extension is **proposed**; Doc-to-LoRA's applicability to code trajectories has not been empirically validated. An Evolution Operator governs the adapter lifecycle using a fitness function combining pass rate and generalization score, performing consolidate, update, forget, and merge operations.

**Expected contribution (~65 words):** If the Phase 1 kill-switch gate succeeds, Rune is **expected** to deliver parametric episodic memory for coding agents — addressing the missing capability identified by Pink et al. The fitness-driven adapter lifecycle is **specified** in the architecture and will be evaluated empirically against LoRA Soups' composition-by-assumption approach. The trajectory modality extension and the PBB-inspired procedural behavior criterion are **proposed** and require empirical validation.
