### Key Findings

- **The autograd break is the single most likely failure mode.** PyTorch forum confirmed: assigning generated weights to `param.data` silently severs the computation graph. The hypernetwork trains with zero gradient. This must be verified with a gradient flow unit test before any other training code runs.

- **Two-pass contamination is invisible in training metrics.** If activations are extracted from the answer-containing pass, the hypernetwork learns to inject answer information rather than context compression. Training loss drops normally — only held-out task evaluation reveals the failure.

- **Qwen3-Coder-Next has 12 attention layers out of 48.** The 3:1 DeltaNet-to-Attention ratio (every 4th layer is GatedAttention) means DeltaNet module names differ from `self_attn.*`. Dynamic discovery is required; hardcoding `num_layers=48` with `self_attn.q_proj` targets will silently find only 12 hook points or produce shape mismatches.

- **`strict=False` weight transfer requires explicit key-presence assertions.** After loading `qwen_4b_d2l` weights, code must assert aggregator keys ARE loaded and head keys ARE NOT — otherwise a silently wrong initialization is mistaken for a successful transfer.

- **Task-ID split must come before augmentation, enforced at the API level.** Splitting after augmentation produces correlated train/test sets; evaluation inflates silently.
