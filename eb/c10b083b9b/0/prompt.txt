## ISSUES

### IMPORTANT: Rate-limit handling in get_paginated()

**Location:** `/Users/noahdolevelixir/Code/rune/libs/model-training/src/model_training/github_client.py`, lines 130-137

**Problem:** The `get_paginated()` method calls `httpx.get()` directly without retry logic. If any page after the first hits a rate-limit 403, the entire operation fails and the caller loses the data already fetched from previous pages.

**Impact:** MODERATE - Reliability issue for mining operations fetching multiple pages

**Fix:** Refactor to use `self.get()` internally instead of calling `httpx.get()` directly:

```python
# Current (line 130):
resp = httpx.get(url, headers=self._headers, params=merged_params, timeout=30.0)

# Should be:
resp = self.get(url, params=merged_params)  # Reuses get()'s retry logic
```

**Effort:** Low (1-2 lines)
