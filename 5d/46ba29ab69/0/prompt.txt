### **11. Rune Agent README**
**Path:** `/Users/noahdolevelixir/Code/rune/services/rune-agent/README.md`

**Project Claims:**
- LangGraph state graph implementing the recursive code generation loop

**Architecture:**
```
START → generate → execute → reflect → [should_retry] → generate (retry) OR save_trajectory → END
```

**Two Graph Modes:**
- `create_graph()` — Standard loop with retry logic (standalone use)
- `create_single_iteration_graph()` — Single iteration: generate → execute → reflect → END (used by `scripts/rune_runner.py` where hypernetwork generates new adapter between iterations)

**Key Files:**
- `graph.py` — Graph construction and `should_retry` router
- `nodes.py` — Node implementations
- `state.py` — `RuneState` TypedDict

**State Tracks:** task description, generated code, execution results, test pass/fail, attempt count, max attempts, trajectory history

**Retry Logic:**
- Tests passed → `save_trajectory` (success)
- Attempts exhausted → `save_trajectory` (exhausted)
- Otherwise → `generate` (retry)

**Relationship to Pipeline:**
- Outer 4-phase pipeline (`scripts/rune_runner.py`) uses `create_single_iteration_graph()` per iteration within a phase
- Between iterations, hypernetwork can generate fresh adapter
- Agent graph handles single cycle; pipeline handles phase sequencing
